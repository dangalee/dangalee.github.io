[1] tailwind css란?

Tailwind CSS is an open source CSS framework. The main feature of this library is that, unlike other CSS frameworks like Bootstrap, it does not provide a series of predefined classes for elements such as buttons or tables. Instead, it creates a list of “utility” CSS classes that can be used to style each element by mixing and matching.
[출처: Wikipedia]

Tailwind CSS는 오픈소스 CSS 프레임워크입니다. Tailwind CSS는 Bootstrap과 같은 CSS 프레임워크와는 달리, 버튼들이나 테이블들을 위한 미리 지정된 class들을 제공하지 않습니다. 대신, 스타일링을 위한 다양한 유틸리티형 CSS 클래스들을 제공합니다.

[2] tailwind css를 선택한 이유

프론트엔드 개발에 입문하며, CSS 프레임워크 결정에 어려움을 느꼈습니다.
기존에 알려진 Bootstrap, Vuetify, Quasar, Element Plus 등을 조사했고, 개인적으로는 Vuetify나 Quasar도 괜찮아 보였으나 팀원들과 의견 조율을 통해 Tailwind CSS로 결정하게 되었습니다. 이와 같은 결정에는 카카오 FE 기술 블로그의 영향도 있었습니다.

카카오 FE 개발그룹에서는 Tailwind CSS를 어떻게 사용했을까?

  • Bootstrap vue: 트위터에서 개발한 CSS 프레임워크로 많은 사용자 층이 형성되어 있다는 것이 장점이나, Bootstrap vue는 Vue3를 미지원하고 정형화된 디자인이 많음

  • Vuetify: Vue3를 지원하는 Vuetify 3.0이 릴리즈 됨 (현재 가장 최신버전은 2023년 11월 v3.4버전), 많은 사용자가 있으나 용량이 크고 무겁다는 단점이 있음

  • Quarsar: Vue3 지원, 사용자 층이 엷다는 것이 단점이나, vuetify보다 다양한 장치에 대한 호환성이 높고 이미 완성된 컴포넌트를 제공함

  • Element Plus: Vue용 프레임워크 중 명성이 높다고 하나 네이버에 생각보다 자료가 없음

  • Tailwind CSS: Utility first 프레임워크로, 기존의 CSS 프레임워크 대비 커스텀이 자유롭다는 장점이 있음.
