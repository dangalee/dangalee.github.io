    FE 개발일기 [3] 개발 컨벤션

    Danga Lee
    2024-03-10

    Goal 💡

    • 프론트엔드 개발 컨벤션
    • 백엔드 개발 컨벤션

    [0] 개인적으로 생각한 개발 컨벤션이 필요한 이유

    • 공동 작업에서 한 명이 작성한 것 같은 코드 작성을 위해서입니다.

    • 코드 리뷰 시 코드 파악이 용이합니다.

    • 컨벤션를 합의하는 과정에서 더 나은 변수명을 설정할 수 있습니다.

    [1] 프론트엔드 개발 컨벤션

    JavaScript

    변수 선언

    • var 대신 const 사용하기
      • 변수명, 함수명
        • 일반 변수 : camelCase
        • const 선언 : 가변값만 let 선언
        • Boolean 변수 : 앞에 is 붙이기
        • 모달 변수: show
      • 상수명
        • const 선언
        • const : UPPER_SNAKE_CASE
      • 클래스, 생성자
        • PascalCase

    String

    • String은 ‘’(작은 따옴표) 사용하기
      • Setting -> quote 검색 -> Prettier: Single Quote 활성화
      • Javascript > Preferences: Quote Style -> single 설정

    함수 선언

    • 가능한 Arrow Function으로 함수식 작성

    [2] 백엔드 개발 컨벤션

    Java

    기본 규칙

    • 인코딩
      • UTF-8
    • 주석
      • 주석: 최대한 상세하게
        ex. 반복문 i, j, k 금지 -> idx

    변수 선언

    • 변수명, 인스턴스명
      • camelCase
    • 상수 변수명 (static final)
      • UPPER_SNAKE_CASE
    • 클래스명, 생성자명
      • PascalCase
    • 함수명
      • camelCase (동사 + 명사로 구성)

    메소드명

    • controller
      • add, get, edit, remove
    • service
      • create, get, update, delete
    • repository
      • jpa rules
